[](#a-定義)[]()A. 定義
----------

**「アフィリエイト」** とは、直接的または間接的に当事者を支配、管理、または当事者と共通の管理下にある法人を意味します。ここで「支配」とは、法人の 50% 超の持分、または経営を支配する権利を有していることを意味します。

**「本契約」** とは、本書に含まれる、または参照されているすべての条項、条件、通知、および GitHub が本サービス上で随時公開するその他すべての運用規則、ポリシー、および手順を総称していいます。 GitHub のサイト ポリシーは、[https://docs.github.com/categories/site-policy](/ja/categories/site-policy) で入手できます。

**「すべてのユーザー」** とは、本サービスを利用するお客様のユーザーおよび外部ユーザーを総称していいます。

**「南北アメリカ」** とは、米国、カナダ、メキシコ、または中南米またはカリブ海諸国の国を意味します。

**「ベータ プレビュー」** とは、アルファ版、ベータ版、プレビュー版、早期アクセス版、または評価版として識別されるソフトウェア、サービス、または機能、または同様の意味を持つ単語または語句を意味します。

**「機密情報」** とは、いずれかの当事者が他方当事者に開示したすべての非公開情報を、書面、口頭またはその他の手段によるかを問わず、機密として指定されているか、または情報開示の環境および情報の性質の下で、受領当事者が知っている、または合理的に知るべきであることを開示当事者にとって機密であることを意味します。 疑義を避けるために付言すると、本サービスに投稿されたコンテンツは、お客様のプライベート リポジトリにのみ保存されているお客様のコンテンツを除き、機密情報とは見なされません。 秘密情報には以下の情報は含まれません。(i) 受領当事者の過失なしに公知となっていた、または公知となった情報、(ii) 開示する権利を有する開示当事者以外の情報源からの機密または専有的制限なしに、受領当事者に正当に知られていたか、または正当に知られることになった情報、(iii) 開示当事者が、当該開示当事者の正当に権限を与えられた役員が署名した書面において、制限なく開示することを承認した情報、(iv) 受領当事者が、他方当事者の機密情報へのアクセスまたは使用なしに独自に開発した情報、または (v) 本サービス上およびお客様のプライベート リポジトリの外部に保存もしくは投稿されている、または保存もしくは投稿されていた情報。

**「コンテンツ」** とは、コード、テキスト、データ、記事、画像、パッケージ、写真、グラフィック、ソフトウェア、アプリケーション、デザイン、機能、および本サービスを通じて紹介、表示、またはその他の方法で利用可能になるその他のマテリアルを意味しますが、これらに限定されません。

**「企業アカウント」** とは、ユーザーが法人に代わって作成したアカウントを意味します。

**「お客様」** とは、[同意する] または同様のボタンをクリックするか、「製品」にアクセスすることにより、GitHub と本契約を締結した会社または組織を意味します。

**「お客様のコンテンツ」** とは、お客様が作成、所有、またはお客様が権利を有するコンテンツを意味します。

**「ドキュメンテーション」** とは、GitHub がお客様に提供または入手可能にする「製品」に関するマニュアル、ドキュメント、およびその他のサポート資料を意味します。

**「発効日」** とは、お客様が (i) 本契約の契約条件に [同意する] をクリックする日、または (ii) 最初に製品を注文した日のいずれか早い方の日付です。

**「外部ユーザー」** とは、お客様のユーザーを除く、本サービスにアクセスする、または本サービスを使用する個人を意味します。

**「フィードバック」** とは、GitHub 製品またはサービスに関するアイデア、ノウハウ、アルゴリズム、コード投稿、提案、機能強化リクエスト、推奨事項、またはその他のフィードバックを意味します。

**「料金」** とは、(i) 該当する期間内に本製品を使用するため、または (ii) プロフェッショナル サービスを受けるために、お客様が GitHub に支払わなければならない料金意味し、かかる料金は注文書または SOW に反映されます。

**「フォーク」** とは、あるリポジトリのコンテンツを別のリポジトリにコピーすることを意味します。

**「GitHub」** とは、GitHub, Inc.、その関連会社、およびその代表者を意味します。

**「GitHub コンテンツ」** とは、GitHub が作成、所有、または GitHub が権利を有するコンテンツを意味します。

**「マシン アカウント」** とは、マシン アカウントに代わって適用されるサービス条件に同意し、有効な電子メール アドレスを提供し、その行動に責任を負う個々の人間によって登録されたアカウントを意味します。 マシン アカウントは、自動化されたタスクを実行するためだけに使用されます。 複数のユーザーがマシン アカウントのアクションを指示できますが、アカウントの所有者はマシンのアクションに対して最終的な責任を負います。

**「注文書」** とは、両当事者が製品を注文するために使用できる書面または電子文書 (見積もりを含む) を意味します。

**「組織アカウント」** とは、1 つの団体または 1 人以上のユーザーに関連付けることができる共有ワークスペースを意味し、複数のユーザーが一度に複数のプロジェクトにわたってコラボレーションすることができます。 ユーザーは、複数の組織アカウントのメンバーになることができます。

**「プライベート リポジトリ」** とは、ユーザーがコンテンツへのアクセスを制御できるリポジトリを意味します。

**「プロフェッショナル サービス」** とは、GitHub が相互に実行される SOW に従ってお客様に提供する研修、コンサルティング、または実装サービスを意味します。 プロフェッショナル サービスには、サポートは含まれません。

**「プロフェッショナル サービス クレジット」** とは、プロフェッショナル サービス (旅費および宿泊費を除く) を購入するための前払いの支払い方法を意味し、お客様はプロフェッショナル サービスのために 12 か月間 (注文書に別段の記載がない限り) 使用することができます。 購入日から 12 か月 (または注文書に別段の記載がある) を過ぎても未使用のままのプロフェッショナル サービス クレジットは、自動的に解約され、返金はできません。

**「パブリック リポジトリ」** とは、コンテンツがすべてのユーザーに表示されるリポジトリを意味します。

**「代表者」** とは、当事者の従業員、役員、代理人、独立請負業者、コンサルタント、ならびに法律顧問および財務顧問を意味します。

**「スクレイピング」** とは、ボットやウェブクローラーなどの自動的な処理により、本サービスからデータを抽出することを意味します。GitHub の API を介した情報の収集は含まれません。

**「サービス」** とは、GitHub のホストされたサービスおよび該当するドキュメンテーションを意味します。

**「SOW」** とは、GitHub が実行するプロフェッショナル サービス、関連する料金、および各当事者の関連する義務を詳述した、相互に実行される作業明細書を意味します。

**「サブスクリプション ライセンス」** とは、お客様に代わって本サービスをインストール、操作、アクセス、および使用するために各ユーザーに割り当てられるライセンスを意味します。 お客様は、その組織アカウント全体でユーザーごとに 1 つのサブスクリプション ライセンスのみを割り当てることができます。 明確にするために記すと、お客様がサブスクリプション ライセンスをユーザーに割り当てた後、お客様がサブスクリプション ライセンスを分割して、あるユーザーがある組織アカウントでサブスクリプション ライセンスを使用し、別のユーザーが別の組織アカウントで同じサブスクリプション ライセンスを使用できるようにすることは禁止されています。

**「サポート」** とは、GitHub が提供するサービスの技術サポートを意味します。

**「ユーザー」** とは、(a) 本サービスにアクセスまたは使用する個人またはマシン アカウント、(b) お客様のアカウントの一部にアクセスまたは使用する個人またはマシン アカウントを意味します。または (c) 機能の実行におけるお客様のアカウントの使用を指示し、いずれの場合もお客様に代わって指示します。 ユーザー数は、お客様が購入したサブスクリプション ライセンスの数を超えてはなりません。

**「ユーザー生成コンテンツ」** とは、第三者または外部ユーザーが作成または所有するコンテンツを意味します。
