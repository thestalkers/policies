### [2. 必需信息](#2-required-information) ###

您必须提供有效的电子邮件地址才能完成注册过程。 要求的任何其他信息（例如您的真实姓名）都是可选的，除非您代表法律实体接受这些条款（在这种情况下，我们需要有关该法律实体的更多信息），或者您选择[付费帐户](#l-payment)（在这种情况下，出于计费目的，需要您提供其他信息）。

您享有对您的内容的所有权。 如果您发布非本人创作的内容，您有责任确保拥有发布该内容的权利，并遵守所有适用的许可。

### [4.向我们授予许可](#4-license-grant-to-us) ###

您授予 GitHub 和我们的关联公司存储、托管、存档、解析、显示和复制您的内容的必要权利，以便提供、开发和改进服务，包括训练 AI 功能，以及训练、开发和改进我们关联公司的人工智能及机器学习模型和技术。 此许可包含的权利有：将您的内容复制到我们的数据库并制作备份；向您及其他用户显示；解析内容并编入搜索索引，或在我们的服务器上进行其它形式的分析；与其他用户共享内容；如果您的内容是音乐或视频，则可以对其进行播放。 为避免疑义，使用您的内容开发、训练和改进 GitHub 及我们关联公司的人工智能及机器学习模型和技术，均在此许可范围内，并且不构成对您内容的销售或其他受限转让。

### [5.向其他用户授予许可](#5-license-grant-to-other-users) ###

您公开发布的内容，包括议题、评论以及对其他用户存储库的贡献，均可被他人查看。 将您的存储库设置为可公开查看，即表示您同意允许他人查看和“复刻”您的存储库（这意味着他人可以在本服务内，从他们控制的存储库自行制作副本）。

通过将存储库设为公开，您授予其他用户一项非排他性、全球许可，可根据 GitHub 的功能权限，通过本服务使用、显示、执行和复制（通过复刻）您的内容。 您可以通过[采用一项许可](/zh/communities/setting-up-your-project-for-healthy-contributions/adding-a-license-to-a-repository#including-an-open-source-license-in-your-repository)授予额外权利。 如果您发布非本人创建或拥有的内容，您有责任确保该内容已依据允许此类使用的条款获得许可。

### [6.存储库许可下的贡献](#6-contributions-under-repository-license) ###

只要您向包含许可通知的存储库添加内容，则表示您在相同的条款下许可该内容，并且同意您有权利在这些条款下许可该内容。 如果您使用单独的协议（如贡献者许可协议）在不同的条款下许可该内容，则该协议优先。
